Contributing Guidelines

Thanks for your interest in contributing! This is a simple project, so the process is straightforward.

How to Contribute
1. Fork the Repository

Create your own copy of the project by clicking the Fork button.

2. Create a Branch

Make a new branch for your changes:
git checkout -b feature-name

3. Make Your Changes

Keep your code clean and readable. Follow the existing structure and style of the project.

4. Test Your Changes

Before submitting, make sure:

The app runs without errors

All core features (add, update, delete tasks) still work

5. Submit a Pull Request

Open a PR with a short description of what you changed and why.

Code Style

Use clear variable and function names

Keep functions small and focused

Follow the existing PHP, JS, and CSS conventions in the project

Reporting Issues

If you find bugs or have suggestions:

Check if the issue already exists

If not, open a new issue with clear steps to reproduce
